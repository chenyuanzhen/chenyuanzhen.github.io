Backward Digit Sums G/S

2021-04-17
题目笔记

P1118 [USACO06FEB]Backward Digit Sums G/S

关键词

暴力枚举, 杨辉三角

分析

题目数据量

可以说大概率是暴力枚举.

题目说要倒着玩该游戏, 谁没事有事倒着玩. 但对于倒着玩真的太难了. 加之数据量少, 可以尝试正着枚举所有情况. 题目要求字典序, 毫无疑问, 枚举要按照一定顺序, 可以使用全排列算法枚举. 题目给出的n = 4. 对于1234, 摆的位置不同, 可能得到的答案也是不同. 如果按照题目的方式模拟, 时间复杂度是O(n2)O(n^2)O(n2). 当然数据量不大, 目前可以接受. 但是, 乘上枚举的数量就不能接受了. 我采用的是C++中next_permutation. 可以理解是产生1234的全排列. 枚举的时间复杂度是O(n!)O(n!)O(n!). 而当n=12时, 枚举次数高达399168003991680039916800次. 然后乘上12∗12∗39916800=574801920012*12 * 39916800 = 574801920012∗12∗39916800=5748019200次. 这谁受得了.

所以目前优化方向有

  1. 优化根据全排序结果计算游戏答案的过程
  2. 减少枚举的数量.

对于方向1. 看看能不能再O(n)O(n)O(n)内完成.遍历完数组就完事.

当n=4时

abcda+bb+cc+da+2b+cb+2c+da+3b+3c+da \quad b \quad c \quad d \\ a + b \quad b + c \quad c + d \\ a + 2b + c \quad b + 2c + d \\ a + 3b + 3c +d abcda+bb+cc+da+2b+cb+2c+da+3b+3c+d

当n=3时

abca+bb+ca+2b+ca \quad b \quad c \\ a + b \quad b + c \\ a + 2b + c abca+bb+ca+2b+c

对于方向2, 其实没有什么想法. 唯一能做的, 即当i≤ni \leq ni≤n时, ans已经超过题目中的s, 即可直接终止本次检查. 这样做本质也不能减少枚举量
